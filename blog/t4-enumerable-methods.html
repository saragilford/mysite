How the Map Method Works

A brief look at enumberable methods.

February 6, 2015

Say we have an array of integers, and we want to add 1 to each integer in our array. We'd want to use #each, right? Well, not so fast. Let's see what happens when we try to use #each.

Womp womp.

So why can't we just use #each? Well, #each will always return your original array. Imagine your array as a bookshelf, and each object in the array as a book.#each takes each book off the shelf and looks at it, but ultimately just puts it back in the same place unaltered. So what method would we use if we wanted to dog-ear page 75 in each book before we put it back on the shelf? In other words, what if we wanted to to get a modified version of an array? Well, that's where #map comes in.

#map takes each object in the array, and runs it through your block of code, and returns a modified array.

A note here: while #map does return a modified version of the array, it doesn't change the array itself. You can still call on it by typing the name of your array.
